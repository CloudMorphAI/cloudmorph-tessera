# v0.4.0 — Add Mistral and Cohere as PolicyAuthor Providers

## Problem

The OSS package's LLM policy-authoring subsystem (`tessera/llm/`, described in `status/llm-policy-authoring.md`) ships 5 providers in v0.2.0: Gemini (primary), Anthropic, OpenAI, Bedrock, Azure OpenAI. Two credible providers are deliberately deferred from v0.2.0:

- **Mistral** — EU-resident-data positioning (`mistral-large-latest` runs on EU-hosted infrastructure). Important for customers who can't send tool catalogs to non-EU endpoints.
- **Cohere** — `command-r-plus` is cost-competitive at the high end of structured-output quality. Stronger token-cost ratio than the OpenAI/Anthropic enterprise tiers for high-volume usage.

The founder's funding stack targets both — Mistralship (€30K cohort) and the Cohere startup program (~25% off enterprise pricing) — but these credits aren't acquired yet as of v0.2.0. Adding the providers before the credits arrive would mean paying full retail; adding them after the credits arrive matches the cost-discipline pattern that drove the original 5-provider choice (Gemini's free-tier API credits + Anthropic/OpenAI/Bedrock/Azure-OpenAI's enterprise-default footprint).

This improvement closes the gap once the credits are in hand.

## Architectural property: invocation-time-optional (unchanged)

Adding new providers does **not** change the firewall hot path. The LLM subsystem is invocation-time-optional — no `tools/call` ever consults Mistral or Cohere. No latency budget, no provider-outage blast radius on the firewall. The work here is strictly authoring-side: two new provider classes implementing `PolicyAuthor` + `ToolCatalogAnalyzer`.

## Scope

### 1. `tessera/llm/mistral.py:MistralPolicyAuthor`

Implements both Protocols. SDK: `mistralai` (`pip install mistralai`). Default model: `mistral-large-latest` (current as of credit acquisition; verify model name at implementation time).

Mistral has native structured-JSON output via the `response_format={"type": "json_object"}` field on the chat-completions endpoint. The provider class mirrors the Gemini implementation's pattern (low temperature, schema-driven system prompt from `_shared.build_system_prompt()`, retry-on-invalid-YAML loop with up to 3 attempts).

### 2. `tessera/llm/cohere.py:CoherePolicyAuthor`

SDK: `cohere` (`pip install cohere`). Default model: `command-r-plus`.

Cohere's structured-output story uses the `response_format` parameter with a JSON schema. The provider class can reuse `_shared.build_system_prompt()` directly; only the SDK invocation differs.

### 3. CLI integration

Update `tessera/cli.py:_resolve_llm_provider` to add two new `model_lower` cases:

## What's intentionally NOT in this improvement

- **Per-provider prompt customization.** All 7 providers (5 existing + 2 new) share `_shared.build_system_prompt()` verbatim. If a provider needs different prompt scaffolding for any reason (e.g., Mistral's tokenizer demands a tighter prompt budget than Gemini's), that's a separate concern. For v0.4.0 the assumption is the shared prompt works.
- **Firewall hot path implications.** Reiterated for clarity: no `predicted_cost`, `blast_radius`, or any other condition consults Mistral or Cohere. The LLM subsystem is policy-authoring-only.
- **Mistral/Cohere as authentication providers.** Out of scope. Tessera consumes JWTs from OIDC providers (Clerk, Auth0, Cognito, etc.); Mistral and Cohere don't issue auth tokens.
- **Default-provider switch.** Gemini remains the v0.4.0 CLI default. Switching defaults is a separate operational decision based on credit allocation; that decision can happen at any time without code changes.

## Cost-discipline default

Each provider class's `__init__` hardcodes the cheapest-credible model as the default. For Mistral that's `mistral-large-latest` (verify model name + pricing tier at implementation time). For Cohere that's `command-r-plus`. Users with stronger requirements override at instantiation: `MistralPolicyAuthor(model="mistral-large-2407")` or `CoherePolicyAuthor(model="command-r")`.

The cost-discipline pattern is implicit in the choice of defaults. Calling-out is in `status/llm-policy-authoring.md` under "Cost-discipline default."

## Acceptance criteria

- `tessera policy author --intent "..." --model mistral` produces valid YAML policies.
- `tessera policy author --intent "..." --model cohere` produces valid YAML policies.
- `tessera analyze --mcp <url> --model mistral` and `--model cohere` both work.
- Unit tests pass without API keys (mocked SDKs).
- Integration tests pass with real API keys when set.
- The "Why these 5 specifically" section in `status/llm-policy-authoring.md` is updated to "Why these 7 specifically."
